---
title: MasumiIdentity
description: "The single entry-point class for the SDK. Instantiate once per environment and reuse."
---

`MasumiIdentity` is the high-level client that wraps the Masumi credential server and KERIA into a small, typed, idiomatic surface.

## Import

```ts
import { MasumiIdentity } from "@masumi_network/identity-sdk";
```

## Constructor

```ts
new MasumiIdentity(config: MasumiIdentityConfig)
```

### `config: MasumiIdentityConfig`

<ParamField body="credentialServerUrl" type="string" required>
  Base URL of the Masumi KERI credential server. Production:
  `https://cred-issuance.masumi-identity.xyz`.
</ParamField>

<ParamField body="keriaUrl" type="string" required>
  Base URL of the KERIA agent. Production: `https://keria.masumi-identity.xyz`.
  Required for signature-verification flows; supply a placeholder if your
  integration never calls `verifyAidSignature` / `fetchKeyState`.
</ParamField>

<ParamField body="fetch" type="typeof fetch">
  Optional custom `fetch` implementation (useful for testing, tracing, or
  non-Node runtimes). Defaults to the global `fetch`.
</ParamField>

### Example

<Tabs>
<TabItem value="canonical" label="Canonical production">

```ts
import {
  MasumiIdentity,
  MASUMI_IDENTITY_ENDPOINTS,
} from "@masumi_network/identity-sdk";

const identity = new MasumiIdentity(MASUMI_IDENTITY_ENDPOINTS.production);
```

</TabItem>
<TabItem value="explicit" label="Explicit URLs">

```ts
import { MasumiIdentity } from "@masumi_network/identity-sdk";

const identity = new MasumiIdentity({
  credentialServerUrl: "https://cred-issuance.masumi-identity.xyz",
  keriaUrl: "https://keria.masumi-identity.xyz",
});
```

</TabItem>
<TabItem value="self-hosted" label="Self-hosted / staging">

```ts
import { MasumiIdentity } from "@masumi_network/identity-sdk";

const identity = new MasumiIdentity({
  credentialServerUrl: process.env.CRED_URL!,
  keriaUrl: process.env.KERIA_URL!,
});
```

</TabItem>
<TabItem value="custom-fetch" label="Custom fetch">

```ts
import { MasumiIdentity } from "@masumi_network/identity-sdk";

const tracedFetch: typeof fetch = async (input, init) => {
  console.log("[SDK]", input);
  return fetch(input, init);
};

const identity = new MasumiIdentity({
  credentialServerUrl: "...",
  keriaUrl: "...",
  fetch: tracedFetch,
});
```

</TabItem>
</Tabs>

## Lifecycle

The client is safe to instantiate once per process and reuse. It holds no mutable state, only the configured base URLs and `fetch` implementation. There is no `close()` or `dispose()` — the underlying `fetch` connections are managed by the runtime.
